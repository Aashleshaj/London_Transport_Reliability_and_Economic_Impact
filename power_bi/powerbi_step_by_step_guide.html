Step-by-step Power BI guide for London Transport dashboard

A
Before you open Power BI — run the Python pipeline
1

Run data_integration.py

Open a terminal in your project folder and run:
python scripts/data_integration.py
This produces 4 output CSVs in the data/ folder.

2

Run correlation_analysis.py

python scripts/correlation_analysis.py
This produces 3 more CSVs including risk scores and cause breakdown.

3

Confirm all 7 files exist in data/

all_lines_combined.csv disruption_enriched.csv borough_disruption_summary.csv disruption_cause_summary.csv mode_disruption_summary.csv forecast_disruption_risk.csv merged_transport_economic.csv

B
Load the 7 CSV files into Power BI
1

Open Power BI Desktop → Get Data

Click Home tab → Get Data → Text/CSV

2

Load all_lines_combined.csv first

Navigate to your data/ folder → select all_lines_combined.csv → click Load (not Transform — the data is already clean).

3

Repeat for the remaining 6 files

Each time: Get Data → Text/CSV → select file → Load. After all 7 are loaded you will see them listed in the Data pane on the right.

If Power BI asks about data type detection, choose "Based on first 200 rows". Do not change column types — they are already correct.
4

Fix the is_disrupted column type

In the Data pane click all_lines_combined → find column is_disrupted → in the column header dropdown change type from Text to True/False. Do the same for disruption_enriched.

C
Build the data model (relationships)
1

Switch to Model view

Click the Model icon on the left sidebar (looks like three connected shapes). You will see all 7 tables laid out.

2

Create these 4 relationships by dragging

Drag the borough field from one table and drop it onto borough in another to create each relationship:

3

Set cross-filter direction for Many:Many

Double-click the line between disruption_enriched and all_lines_combined → set Cross filter direction to Both → click OK.

D
Create the 6 DAX measures
1

Right-click all_lines_combined → New measure

In the Data pane, right-click the all_lines_combined table → New measure. A formula bar appears at the top. Paste each measure below and press Enter.

Disruption Rate % = DIVIDE( COUNTROWS(FILTER('all_lines_combined', 'all_lines_combined'[is_disrupted] = TRUE())), COUNTROWS('all_lines_combined'), 0 ) * 100
Avg Severity = AVERAGE('all_lines_combined'[statusSeverity])
Total Disruptions = COUNTROWS('disruption_enriched')
Total GVA at Risk £m = SUM('forecast_disruption_risk'[estimated_gva_at_risk_m])
Severity Band = SWITCH(TRUE(), [Avg Severity] >= 9, "Good Service", [Avg Severity] >= 7, "Minor Issues", [Avg Severity] >= 5, "Moderate", "Severe" )
Total Lines Monitored = DISTINCTCOUNT('all_lines_combined'[name])
E
Create Page 1 — Network overview
1

Rename the first page

Right-click the tab at the bottom → Rename page → type Network Overview

2

Add 4 KPI cards along the top

In Visualizations pane click Card → drag it to the top-left. In the Fields pane drag in [Disruption Rate %]. Repeat for 3 more cards: [Total Disruptions], [Avg Severity], [Total GVA at Risk £m]. Arrange them in a row.

3

Add a Stacked Bar Chart — lines by mode and status

Click Stacked bar chart visual → set:
X-axis: all_lines_combined[modeName]
Y-axis: Count of all_lines_combined[name]
Legend: all_lines_combined[severity_label]

4

Add a Donut Chart — severity mix

Click Donut chart → set:
Legend: all_lines_combined[severity_label]
Values: Count of all_lines_combined[name]

5

Add a Table — all lines detail

Click Table visual → drag in:
name, modeName, severity_label, borough, is_disrupted, reason (all from all_lines_combined)

F
Create Page 2 — Disruption analysis
1

Add a new page → name it "Disruption Analysis"

Click the + at the bottom → right-click → Rename

2

Add a horizontal Bar Chart — disruptions by cause

Click Bar chart → set:
Y-axis: disruption_cause_summary[disruption_cause]
X-axis: disruption_cause_summary[incidents]
Color: disruption_cause_summary[avg_severity] → in Format pane set Diverging color scale (red = low severity score)

3

Add a Bar Chart — disruption rate by mode

Click Clustered bar chart → set:
X-axis: mode_disruption_summary[modeName]
Y-axis: mode_disruption_summary[disruption_rate_pct]
Color: mode_disruption_summary[avg_severity]

4

Add a Matrix — borough vs cause

Click Matrix visual → set:
Rows: disruption_enriched[borough]
Columns: disruption_enriched[disruption_cause]
Values: Count of disruption_enriched[name]

5

Add a Table — disruption detail rows

Click Table → drag in from disruption_enriched:
name, modeName, severity_label, disruption_cause, affected_section, reason, borough

6

Add a Slicer — filter by severity

Click Slicer → drag in all_lines_combined[severity_label]. In Format pane change style to Dropdown.

G
Create Page 3 — Economic impact
1

Add a new page → name it "Economic Impact"

Click + → rename

2

Add Scatter Chart — GVA vs severity

Click Scatter chart → set:
X-axis: borough_disruption_summary[total_gva_m]
Y-axis: borough_disruption_summary[average_severity]
Size: borough_disruption_summary[total_employees]
Legend / Details: borough_disruption_summary[borough]
Color saturation: borough_disruption_summary[disruption_rate_pct]

3

Add Scatter Chart — employment vs disruption rate

Click Scatter chart → set:
X-axis: borough_disruption_summary[total_employees]
Y-axis: borough_disruption_summary[disruption_rate_pct]
Details: borough_disruption_summary[borough]

4

Add a Bar Chart — GVA at risk by borough

Click Clustered bar chart → set:
X-axis: forecast_disruption_risk[borough]
Y-axis: forecast_disruption_risk[estimated_gva_at_risk_m]
Legend: forecast_disruption_risk[risk_band]

5

Add a Table — borough economic summary

Click Table → drag in from borough_disruption_summary:
borough, total_gva_m, total_employees, total_sales_m, disruption_rate_pct, average_severity

H
Create Page 4 — Forecast & risk
1

Add a new page → name it "Forecast and Risk"

Click + → rename

2

Add Bar Chart — composite risk score by borough

Click Clustered bar chart → set:
X-axis: forecast_disruption_risk[borough]
Y-axis: forecast_disruption_risk[composite_risk_score]
Legend: forecast_disruption_risk[risk_band]

3

Add Line Chart — 5-year forecast sales

First unpivot the forecast columns in Power Query: Transform Data → select the 5 forecast_sales columns → Unpivot columns. This gives you an Attribute (year) and Value (sales £m) column.

Then in the visual: X-axis = Attribute, Y-axis = Value, Legend = borough

4

Add a Gauge — risk score for selected borough

Click Gauge → set:
Value: forecast_disruption_risk[composite_risk_score]
Min value: 0, Max value: 1
Target value: 0.5 (medium threshold)
This responds to the borough slicer on the page.

5

Add a Slicer — risk band filter

Click Slicer → drag in forecast_disruption_risk[risk_band]. Change style to Tile in Format pane for easy click-through filtering.

I
Add slicers to every page
1

Add these 5 slicers to each page

borough_disruption_summary[borough]
all_lines_combined[modeName]
all_lines_combined[severity_label]
disruption_enriched[disruption_cause]
forecast_disruption_risk[risk_band]

2

Sync slicers across pages

Go to View → Sync slicers. For each slicer, tick all 4 page columns so selections carry across pages. This lets filtering by borough on Page 1 automatically filter all other pages.

J
Apply the colour theme
Good Service / Very Low risk#27AE60
Minor Delays / Medium risk#F39C12
Severe Delays / High risk#E74C3C
Suspended / Critical risk#7F0000
1

Apply colours to severity_label in each bar/donut chart

Select a chart → Format pane → Data colors → fx (conditional formatting) → set rules: if severity_label = "Good Service" → hex #27AE60, "Minor Delays" → #F39C12, "Severe Delays" → #E74C3C, "Suspended" → #7F0000

2

Apply the same logic to risk_band colours

Wherever risk_band is used as a legend: Very Low = #27AE60, Low = #2ECC71, Medium = #F39C12, High = #E74C3C, Critical = #7F0000

K
Save and publish
1

Save the .pbix file

File → Save as → LondonTransportDashboard.pbix

2

Publish to Power BI Service (optional)

Home → Publish → select your workspace. Once published, you can share via URL and set up scheduled refresh.

3

Set up scheduled refresh (optional)

In Power BI Service: open the dataset → Settings → Scheduled refresh → On. Set frequency (daily recommended). You must also install a Power BI Gateway on the machine where your CSV files live.
